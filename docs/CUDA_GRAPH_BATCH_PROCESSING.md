# CUDA Graph 批处理测试文档

## 一、本次测试包括的内容

根据 `benchmark()` 函数代码，测试分为三个阶段：

### 1. H2D Transfer (Host → Device) - 0.929 ms (25%)
- **图像数据传输**: 原始图片 RGB 数据 → GPU
- **ImageInfo 传输**: 图片尺寸信息 → GPU
- 使用 **Pinned Memory** (锁页内存) 加速传输

### 2. GPU Pipeline - 2.737 ms (73%) - CUDA Graph 核心部分

包含三个子阶段，全部封装在 CUDA Graph 中：

| 子阶段 | 功能 |
|--------|------|
| **预处理** | Letterbox缩放、双线性插值、RGB→BGR、归一化(/255) |
| **推理** | TensorRT 引擎执行 (FP16) |
| **后处理** | 输出解码、置信度过滤、NMS、坐标缩放回原图 |

### 3. D2H Transfer (Device → Host) - 0.073 ms (2%)
- **检测结果传输**: PoseResult (bbox + 17个关键点)
- **检测数量传输**: 每张图的检测人数

### 未包含在测试中的部分
- 图片加载 (stb_image)
- 结果绘制 (draw_box, draw_keypoints)
- 图片保存 (stbi_write_jpg)
- 窗口显示

### 性能分布 (batch=4)

| 阶段 | 时间 | 占比 |
|------|------|------|
| H2D Transfer | 0.93 ms | 25% |
| **GPU Pipeline** | **2.74 ms** | **73%** |
| D2H Transfer | 0.07 ms | 2% |
| **Total** | **3.74 ms** | 100% |

**结论**: GPU Pipeline 占 73%，是主要瓶颈，其中 TensorRT 推理是最大开销。

## 二、预处理详细内容

### 1. Letterbox 缩放
   - 计算缩放比例：保持宽高比缩放到 640x640
   - 计算填充区域：居中放置

### 2. 双线性插值
   - 对每个像素进行双线性插值采样
   - 支持任意尺寸输入图像

### 3. 归一化
   - RGB 通道分离 (BGR → RGB)
   - 归一化：`(val - mean) / std` = `val / 255.0`

### 4. 填充
   - 缩放后的空白区域填充为 0（黑色）

### 5. 元数据保存
   - 记录原始尺寸、缩放比例、填充偏移
   - 用于后处理时坐标还原

## 三、后处理详细内容

### 1. 解码输出 (`decode_output_kernel`)
- **置信度过滤**: 过滤低于阈值 (0.25) 的检测
- **边界框解码**: 从 (cx, cy, w, h) 转换为 (x1, y1, x2, y2)
- **关键点提取**: 提取 17 个关键点 (每个关键点 3 个值: x, y, conf)

### 2. NMS 非极大值抑制 (`nms_kernel`)
- **IoU 计算**: 计算边界框重叠度
- **抑制逻辑**: 
  - 遍历所有检测框
  - 如果有更高分数的框 IoU > 阈值，则抑制当前框
- **保留索引**: 记录未被抑制的检测索引

### 3. 收集结果 (`gather_results_kernel`)
- 按保留索引收集最终结果
- 组装 `PoseResult` 结构体

### 4. 坐标还原 (`scale_coords_kernel`)
- **边界框还原**: 
  ```
  x = (x - pad_x) * scale_x
  y = (y - pad_y) * scale_y
  ```
- **关键点还原**: 同样的缩放逻辑
- 将 640x640 坐标转换回原始图像坐标

## 四、完整流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                        完整推理流程                              │
├─────────────────────────────────────────────────────────────────┤
│  H2D Transfer (0.93ms)                                          │
│  ├── 原始图像数据 (RGB)                                          │
│  └── ImageInfo (src_width, src_height)                          │
├─────────────────────────────────────────────────────────────────┤
│  GPU Pipeline (2.74ms) - CUDA Graph 封装                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 预处理 (preprocess_with_infos_kernel)                    │   │
│  │ ├── Letterbox 缩放 (保持宽高比)                           │   │
│  │ ├── 双线性插值                                          │   │
│  │ ├── RGB归一化 (/255.0)                                   │   │
│  │ └── 填充 (黑色)                                          │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ TensorRT 推理                                            │   │
│  │ └── YOLO11-Pose 模型 (FP16)                              │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ 后处理                                                    │   │
│  │ ├── decode_output_kernel (解码 + 置信度过滤)              │   │
│  │ ├── nms_kernel (非极大值抑制)                             │   │
│  │ ├── gather_results_kernel (收集结果)                      │   │
│  │ └── scale_coords_kernel (坐标还原)                        │   │
│  └─────────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────────┤
│  D2H Transfer (0.07ms)                                          │
│  ├── 检测结果 (PoseResult)                                      │
│  └── 检测数量                                                    │
└─────────────────────────────────────────────────────────────────┘
```

**关键点**: 整个 GPU Pipeline 被封装在 CUDA Graph 中，实现零开销执行，这是性能提升的关键。
